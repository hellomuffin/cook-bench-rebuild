Procedurally scaling task complexity

n_ops × map

Each task is procedurally generated and rated on two axes. n_ops = the minimal number of high-level VLM commands to finish the dish (planning depth). map complexity = a single navigation + grounding score, map = 1·path + 6·grounding_ambiguity — where path is the tiles the cook must walk (it already absorbs obstacle detours & how far apart stations sit) and grounding is the number of duplicate look-alike stations to tell apart. Crucially, map is scaled without growing the grid past 10×10 (so instances stay legible): we add obstacles, spread stations apart, and add duplicate stations. Each cell plays a completion video (a scripted solver finishing the dish) and lists the full ground-truth plan in the VLM action space.
