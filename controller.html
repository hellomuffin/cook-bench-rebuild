Streaming-VLM controller — results

Can a streaming VLM drive the cook?

The VLM acts as the controller: every ~second it sees only the rendered RGB and issues one mid-level command (go to / pick up / scoop / serve …); success = the dish is delivered. We ran gemini-3.1-pro-preview (sliding-window) over the 9 procedurally-generated tasks (n_ops × map), under five input conditions, 3 repeats each (270 runs total incl. the live baseline). Pure-pixel input; full per-decision history logged.

Effect of input condition (Gemini 3.1 Pro, /27)

basic = rendered RGB only · graph_state = labelled top-down state map · agent+ = + per-step action log · agent++ = + per-step location · agent+++ = + plate contents (text).

State helps, monotonically. Pure RGB solves only 7%; adding state climbs to 33% (≈4.5×). The state map alone (19%) nearly triples pure RGB — i.e. a large part of the difficulty is perception, not planning.
